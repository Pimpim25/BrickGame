# BrickGame
Реализация игр «Змейка» и «Тетрис» с поддержкой десктопного и консольного интерфейсов.

# «Змейка»:
- проект состоит из двух отдельных компонентов: библиотеки, отвечающей за реализацию логики игры , и десктопного графического интерфейса на базе Qt. Разработанная библиотеку поддерживается консольным интерфейсом.
- Покрытие библиотеки, реализующей бизнес-логику игры, тестами с помощью библиотеки GTEST.
- библиотека и десктопный графический интерфейс разработаны на языке C++ стандарта C++20 с использованием компилятора gcc.

# «Тетрис»:
- проект состоит из двух отдельных компонентов: библиотеки, отвечающей за реализацию логики игры, и консольног интерфейса.Разработанная библиотеку поддерживается десктопным графическим интерфейсом.
- Покрытие библиотеки, реализующей бизнес-логику игры, тестами с помощью библиотеки check.h.
-  библиотека и консольный интерфейс разработаны на языке С стандарта C11 с использованием компилятора gcc.

# игры разработаны с использованием паттерна КА и MVC. Для взаимодействия интерфейса и библиотеки с логикой игры используются следующие функции:
- Функция `userInput` принимает на вход пользовательское действие `action` и дополнительный параметр `hold`, который отвечает за зажатие клавиши.
- Функция `updateCurrentState` предназначена для получения данных для отрисовки в интерфейсе. Она возвращает структуру, содержащую информацию о текущем состоянии игры. Например, для тетриса истечение таймера приводит к смещению фигуры вниз на один ряд. Эта функция должна вызываться из интерфейса с некоторой периодичностью для поддержания интерфейса в актуальном состоянии.

- Проект включает в себя Makefile со стандартными целями.
